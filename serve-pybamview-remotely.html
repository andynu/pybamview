Serving PyBamView remotely

PyBamView is run as a Flask application, which starts up a mini web application. This means that as well as using PyBamView to browse alignments locally on your own computer, you can also serve PyBamView at a URL that is visible to remote collaborators for easy sharing of alignment views.

Serving PyBamView from your own server

  1. Install PyBamView by following the installation instructions.
  2. Determine the hostname of your server {% highlight bash %} hostname -f {% endhighlight %}
  3. Run PyBamView, following the usage instructions. Use the option "--ip 0.0.0.0", which will allow a remote user to view the results. Also set which port to serve the web application by setting "--port $PORT" where $PORT is some large number, between 1024-65535. The default port for PyBamView is 5000. An example command is shown below: {% highlight bash %} pybamview --bamdir .. --ref hg19.fa --ip 0.0.0.0 --port 5000 {% endhighlight %}
  4. Navigate to http://$HOSTNAME:$PORT in a web broswer, where $HOSTNAME is the hostname mentioned above, and $PORT is the argument you used for the port. You can share this URL, or navigate to an interesting alignment view and copy the resulting URL from the browser, with another person that has access to the same server, they can use this same URL. Note that if you need VPN to access the server, the other user will probably also need VPN access to view the URL.

Serving PyBamView from an Amazon EC2 instance

You may want to host a PyBamView instance somewhere besides your own server, for instance if you want to keep it running indefinitely to publicly host your alignment data. A good option for doing this is through Amazon Web Services.

There are existing tutorials online (e.g. http://blog.garethdwyer.co.za/2013/07/getting-simple-flask-app-running-on.html) for how to serve a Python Flask application using Amazon EC2. Below are specific instructions to go from a fresh Amazon EC2 instance to a URL where you can share PyBamView alignments. The instructions assume you have an AWS account and know how to launch EC2 instances. For more info, check out http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/LaunchingAndUsingInstances.html.

  1. Start up an EC2 instance running Ubuntu through your Amazon Web Services console.

    Note, when choosing what type of instance to run, you'll want to make sure you have enough disk space to store your alignments and reference genome. If you have very high coverage data you might also need to take into account how much memory you will need. See this comparison of EC2 instance types to help you decide which type to use. A micro instance was sufficient for the example served from this site.

    You should also make sure your security group is configured to allow http connections. You can add rules to a security group in the "Security Groups" section of the AWS console, under the "Network & Security" options. Choose the security group you plan to use (or create a new one). Then go to the "Inbound" tab and click "Edit". Click "Add Rule" and select "HTTP" as the type. If you do not do this step, you won't be able to go to the URLs described below.

  2. Log into your EC2 instance through SSH. This usually goes something like this: {% highligh bash %} ssh -i $my_key.pem ubuntu@$ec2-XX-XXX-XXX-XXX.compute-1.amazonaws.com {% endhighlight %} where $my_key.pem is your key pair.
  3. Load all of your data, including any alignments you will want to view plus an option reference genome fasta file, onto the instance. You can store data in the "/mnt/" directory. You can upload data to the instance using scp, wget, or directly from Amazon S3 using s3cmd or similar tools. I assume you have a way to do this and don't go into more details here.
  4. Run: {% highlight bash %} sudo apt-get update {% endhighlight %} to update package lists.
  5. Install what you will need for running Python and using apache: {% highlight bash %} sudo apt-get install python-dev python-pip g++ libapache2-mod-wsgi httpd apache2 libapache2-mod-python libapache2-mod-wsgi {% endhighlight %}
  6. Use pip to install pybamview {% highlight bash %} pip install pybamvie {% endhighlight %}
  7. Edit /etc/apache2/httpd.conf to look something like this: {% highlight html %} <VirtualHost *:80> ServerName mydomain.com ProxyPass / http://localhost:5000/ <Location /> Order allow,deny allow from all deny from none <Location> </VirtualHost> {% end highlight %}
  8. Run: {% highlight bash %} sudo a2enmod proxy_balancer sudo service apache2 restart {% endhighlight %}
